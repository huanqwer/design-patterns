# 贡献指南

感谢您对本项目的关注！我们欢迎任何形式的贡献。

## 如何贡献

### 报告问题

如果您发现了bug或有功能建议，请通过以下方式联系我们：

- 发送邮件至：huanqwer0314@gmail.com

### 提交代码

我们欢迎各位开发者使用不同的编程语言（Go、Python、PHP、C++等）来实现33种设计模式，以丰富本项目的内容。

#### 代码规范

- 请确保代码清晰、易读
- 添加必要的注释说明
- 遵循现有代码的目录结构

#### 提交流程

1. Fork 本项目
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个 Pull Request

## 设计模式列表

本项目致力于实现33种经典设计模式，包括：

- **创建型模式**：工厂方法模式、抽象工厂模式、单例模式、建造者模式、原型模式
- **结构型模式**：适配器模式、装饰器模式、代理模式、外观模式、桥接模式、组合模式、享元模式
- **行为型模式**：策略模式、模板方法模式、观察者模式、迭代子模式、责任链模式、命令模式、备忘录模式、状态模式、访问者模式、中介者模式、解释器模式
- **其他模式**：MVC模式、业务代表模式、组合实体模式、数据访问对象模式、前端控制器模式、拦截过滤器模式、服务定位器模式、传输对象模式
